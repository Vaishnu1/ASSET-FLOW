## Version Description

>Versioning of the application is represented in 4 digits as explained below. Each digit signifies the impact of changes implemented.

**`<Major>.<Minor>.<Refresh>-<Build>`**

| Version | Description |
|---------|-------------|
| Major | Major version signifies that the version has new major features which may not be compatible with previous versions.|
| Minor | Minor version signifies that the version has new minor features and bug fixes and mostly compatible with previous versions.|
| Refresh | Refresh version signifies only bugfixes or hotfixes.|
| Build | Running build number.|
